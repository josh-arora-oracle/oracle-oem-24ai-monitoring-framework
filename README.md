# Oracle OEM 24ai Monitoring Framework
Overview

This repository contains a reusable Oracle Enterprise Manager (OEM) 24ai automation framework designed for:

CPU spike forensic classification

Automated health check reporting

Tablespace usage monitoring

Executive-style HTML report generation

Secure wallet-based repository connectivity

All files are sanitized for public release. No client-identifiable data is included.

Source Code

The primary automation logic is implemented in the following shell scripts:

cpu_forensics_nonprod_ACE_SANITIZED.sh

oem_report_dev_ACE_SANITIZED.sh

These scripts query the OEM repository using SQL*Plus and generate structured HTML reports suitable for operational review and capacity planning.

Components
1. CPU Forensics Script

Generates structured HTML reports analyzing CPU usage spikes over a defined lookback window.

Features:

Historical spike classification logic

Threshold-based analysis

Structured executive formatting

Automated HTML output

Designed for non-production scope but adaptable for production environments.

2. OEM Health Check Script

Generates HTML-based health summaries including:

Database utilization indicators

Tablespace threshold analysis

Target monitoring summaries

Capacity signals

The output is formatted for operational review and infrastructure reporting workflows.

Modular script structure for extensibility
